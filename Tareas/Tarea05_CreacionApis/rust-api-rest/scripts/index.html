Práctica: Creación de API REST con Rust

Una guía completa sobre el desarrollo de un API para gestionar Alumnos, usando Rust, MySQL y herramientas modernas.

Presentado por Grok (basado en la práctica anterior).

Objetivo de la Práctica

  • Crear un API REST para el recurso "Alumnos".
  • Endpoints: GET (listar/obtener), POST (crear), PUT (actualizar), DELETE (eliminar).
  • Integrar base de datos MySQL para persistencia de datos.
  • Documentar el API con Swagger (OpenAPI) para pruebas interactivas.
  • Desarrollar en VS Code con Rust.

Tecnologías Principales

  • Rust: Lenguaje de programación seguro, rápido y concurrente. Versión usada: 1.89.0.
  • MySQL: Base de datos relacional para almacenar datos de alumnos (tabla con id, nombre, edad).
  • Desarrollo en VS Code con extensiones como Rust-Analyzer para autocompletado y análisis de código.

Rust destaca por su seguridad de memoria sin garbage collector, ideal para APIs de alto rendimiento.

Frameworks y Librerías Usadas

  • Actix-web (4.11.0): Framework web async para manejar rutas y servidores HTTP.
  • SQLx (0.8.6): Librería para consultas async a MySQL, seguras y verificadas en compile-time.
  • Utoipa (5.4.0) + Utoipa-swagger-ui (9.0.2): Para generar documentación OpenAPI y servir Swagger UI.
  • Serde (1.0.219): Serialización/deserialización de JSON.
  • Dotenv (0.15.0): Cargar variables de entorno (e.g., conexión DB).
  • Chrono (0.4): Manejo de fechas (opcional en el modelo).
  • Tokio (1): Runtime async para ejecutar el servidor.

Estas librerías forman un ecosistema eficiente y moderno en Rust para APIs.

Comandos para Crear y Configurar el Proyecto

cargo new rest_api_alumnos
cd rest_api_alumnos

Edita Cargo.toml con las dependencias listadas.

cargo build  // Descarga e instala dependencias

Crea .env con DATABASE_URL=mysql://root:tu_password@localhost:3306/alumnos_db

Configura MySQL:

sudo mysql -u root -p
CREATE DATABASE alumnos_db;
USE alumnos_db;
CREATE TABLE alumnos (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(255) NOT NULL,
    edad INT NOT NULL
);

Comandos para Ejecutar y Probar

cargo run  // Inicia el servidor en http://localhost:8080

Accede a Swagger: http://localhost:8080/swagger-ui/

Pruebas con curl:

curl -X POST http://localhost:8080/alumnos -H "Content-Type: application/json" -d '{"nombre":"Ana","edad":22}'
curl http://localhost:8080/alumnos
curl http://localhost:8080/alumnos/1
curl -X PUT http://localhost:8080/alumnos/1 -H "Content-Type: application/json" -d '{"nombre":"Ana Actualizada","edad":23}'
curl -X DELETE http://localhost:8080/alumnos/1

Otros comandos: cargo test (para tests), cargo fmt (formateo), cargo clippy (linting).

Partes Importantes de Rust en Esta Práctica

  • Ownership y Borrowing: Usado en handlers para manejar datos sin copias innecesarias, asegurando seguridad de memoria.
  • Async/Await: Todo el código es async con Tokio y Actix-web para manejo eficiente de I/O (e.g., consultas DB).
  • Macros y Traits: SQLx usa macros para consultas verificadas; Serde con Derive para serialización.
  • Error Handling: Uso de Result y match para manejar errores de DB y HTTP.
  • Cargo: Gestor de paquetes integral para builds y dependencias.
  • Seguridad: Borrow Checker previene data races; SQLx evita inyecciones SQL.

Rust reduce bugs en runtime, haciendo el API confiable y performant.

Estructura del Código

  • Cargo.toml: Dependencias.
  • .env: Configuración DB.
  • src/main.rs: Modelos (Alumno), Handlers (get, post, etc.), Rutas, Conexión DB, OpenAPI.

El servidor usa Actix-web para rutas en /alumnos y Swagger en /swagger-ui.

Conclusión y Extensiones

Esta práctica demuestra cómo Rust facilita APIs seguras y eficientes.

  • Extensiones: Añadir autenticación (JWT), paginación, validaciones.
  • Deploy: Usa Docker o Heroku para producción.
  • Aprendizaje: Lee "The Rust Book" para más profundidad.
